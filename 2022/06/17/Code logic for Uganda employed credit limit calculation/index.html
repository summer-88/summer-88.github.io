2.Determine if the interface request data is empty

3.Calculating the credit limit

3.1 Determining whether the customer is an overdue user of lipa later

​ Check the MusoniLoan loan status by id_number and phone_number. if loan_status=Closed (written off) or in_arrears=true, then end the process with the response message.

{
"limit_status": "limit_rejected",
"limit_amount": 0.0,
"credit_option": "rejected",
"approval_type": "auto_rejection",
"approval_date": null,
"limit_description": "Customer has lipalater loans in arrears",
"has_lipalater_arrears": true
}

3.2 Determining whether the customer has monthly repayment capacity

Determine if the field net_income of the request less than the field expenses of the request, then end the process with the response message.

{
"limit_status": "limit_rejected",
"limit_amount": 0.0,
"credit_option": "rejected",
"approval_type": "auto_rejection",
"approval_date": null,
"limit_description": "Customer income is lower than expenses",
"has_lipalater_arrears": false
}

3.3 Determining whether a consumer is employed by a private sector

Field employee_sector must be private_sector. if not, then end the process with the response message.

{
"limit_status": "limit_inactive",
"limit_amount": 0.0,
"credit_option": "pending_approval",
"approval_type": "",
"approval_date": null,
"limit_description": "Customer is employed in public sector",
"has_lipalater_arrears": false
}

3.4 Determining whether a customer paid via cash or mobile money

Field payment_type must be cash or mobile_money. if not, then end the process with the response message.

{
"limit_status": "limit_extra_docs",
"limit_amount": 0.0,
"credit_option": "pending_approval",
"approval_type": "",
"approval_date": null,
"limit_description": "Customer is not paid via cash or mobile money",
"has_lipalater_arrears": false
}

3.5 Get Credit Reference Bureau Report

If net_income >=75000 get json_report.

If api_code == ‘E017’ then return this report.

{
	"credit_score": 0,
	"ppi_score": 'NR',
	"accounts_size": 0,
	"performing_accts": 0,
	"npa_accounts": 0,
	"pwd_accounts": 0,
	"credit_mix": 0,
	"has_error": false
}

If get CRB report has error then return this report.

{
	"credit_score": 0,
	"ppi_score": 0,
	"accounts_size": 0,
	"performing_accts": 0,
	"npa_accounts": 0,
	"pwd_accounts": 0,
	"credit_mix": 0,
	"has_error": true
}

Others return this report.

{
	credit_score: crb_data['credit_score'],
	ppi_score: crb_data['ppi_analysis']['ppi_rank'],
	accounts_size: account_size,
	performing_accts: performing_accts.sum,
	npa_accounts: account_npa.sum,
	pwd_accounts: pwd_accounts.sum,
	credit_mix: account_mix
}

If net_income < 75000 get delinquency_report.return this report.

{
	api_code: resp_data['api_code'],
	api_code_description: 	resp_data['api_code_description'],
	delinquency_code: resp_data['delinquency_code'],
	has_error: resp_data['has_error'],
	identity_number: @customer_kyc[:id_number],
	identity_type: '001',
	loan_amount: 1000
}

If get CRB report has error then return this report.

{
	api_code: '',
	api_code_description: '',
	delinquency_code: '001',
	has_error: true,
	identity_number: @customer_kyc[:id_number],
	identity_type: '001',
	loan_amount: 1000
}

3.6 credit worthiness formula

​ Calculate (net_income - expenses) * (85/100) and net_income* (15/100) take whichever is the smallest (the underlined value is the default value and can be configured).

3.7 Calculating the credit limit

End the process with the response message.

{
"limit_status": "limit_inactive",
"limit_amount": credit_worthiness * multiplier * 12 / INTEREST_MULTIPLIER_NIGERIA,
"credit_option": "pending_approval",
"approval_type": "",
"approval_date": null,
"limit_description": "Limit calculated based on DTIR and CRB",
"has_lipalater_arrears": false
}

3.8 The credit limit is rounded to the nearest 1000 and compared with the limit to the smallest value
